# OpenQR

A free, feature-rich QR code generator with full visual customization. Generate, style, and download QR codes as high-resolution PNG or scalable SVG — no account, no tracking, no middleman.

## How the App Works

OpenQR is a fully client-side Next.js application. QR codes are generated and rendered directly in the browser — no data is sent to any server.

**Key features:**
- Customizable dot styles: Square, Rounded, Dots, Diamond, Star, Lines
- Customizable corner and eye styles
- Foreground/background colors with gradient support (linear or radial)
- Logo overlay with size, padding, and corner radius controls
- Error correction levels (L / M / Q / H)
- Download as PNG (2048×2048, print-ready) or SVG (vector, logo not included)
- 7 quick presets + a random style generator
- Dark mode support

**Stack:** Next.js 15, React 19, TypeScript, Tailwind CSS v4, `qrcode` library for matrix generation, Canvas 2D API for rendering.

**Architecture:**
- `src/components/QRGenerator.tsx` — all UI and state
- `src/lib/qr-renderer.ts` — canvas rendering and SVG export logic
- `src/lib/analytics.ts` — optional Umami analytics hook (disabled by default)

QR codes generated by this app encode URLs directly. They work forever without any server dependency.

## Run Locally

**Prerequisites:** Node.js 18+ and npm.

```bash
# Clone the repo
git clone <your-repo-url>
cd openqr

# Install dependencies
npm install

# Start the development server
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) in your browser.

**Other commands:**

```bash
npm run build    # Build for production
npm start        # Start production server (after build)
npm run lint     # Run ESLint
```

**Optional — Umami Analytics:**

The app has an analytics hook that is disabled by default. To enable it, uncomment the script tag in `src/app/layout.tsx` and set `data-website-id` to your Umami site ID.

No environment variables are required to run the app.

## Deploy on Railway

[Railway](https://railway.app) can deploy this app directly from a GitHub repository.

### Steps

1. Push your code to a GitHub repository.

2. Go to [railway.app](https://railway.app) and create a new project.

3. Select **Deploy from GitHub repo** and connect your repository.

4. Railway will auto-detect Next.js and configure the build. No manual setup is needed.

5. Once deployed, Railway will provide a public URL for your app.

### Manual configuration (if needed)

If Railway does not auto-detect the settings, set these in the service configuration:

| Setting | Value |
|---|---|
| Build command | `npm run build` |
| Start command | `npm start` |
| Port | `3000` |

No environment variables are required unless you are enabling Umami analytics, in which case no server-side variables are needed — the Umami script ID is embedded in the HTML.

### Custom domain

In Railway, go to your service → **Settings** → **Domains** to attach a custom domain and Railway will handle TLS automatically.
